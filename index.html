FSAE Dynamics

Developed new A-Arm design for Rear Lower and Upper Suspension

As the 2022 season came to an end for the Columbia FSAE team, it became clear that the rear suspension system proved to be unreliable and extremely hard to manufacture. I lead the design of the new 2023 Lower and Upper Rear A-Arm for 2023 season. The main objective in mind was ease of manufacturing, reliability, and good strength. The previous year's design consisted of a tube that was way too big for the ball bearings, so during the manufacturing process, the tube not only had to be welded onto the bearing, but also smushed to actually get a reputable weld surface area. This year's design matches the length of the steel tube exactly with the ball bearing holder. I also designed the jig alongside this a-arm design. The jig provided easy welding surface and also provided a way to ensure that the a-arm was straight.

Amazon Scraper

Full Stack Development

After starting an e-commerce business on Amazon, it became clear to me that I needed a more efficient way to collect analytics on competitor products. I created a web scraper that will collect data on a product and display it in a clean and easy to read UI. I developed the front end part of the website using React, which provided me flexibility with search bars and other UI elements. I used Node.js and Express.js for my backend framework. Since I was going to be hosting this on a server, I wanted some light firewall protection for my website. I used NGINX to run a reverse proxy to my nodejs server. Expressjs allowed me to filter out different functions for different requests using its middleware flexibility. I also implemented a websocket to provide real time details of the scraping process. To scrape Amazon, I relied primarily on Puppeteer and Cheerio. Puppeteer simulates a human navigating a browser and has various masking features to prevent detection from Amazon. Cheerio was mainly used to parse the HTML and extract the data I needed. As for data storage, I relied on an SQL server. An SQL server offered me powerful query searches as well as easy implementation with nodejs. Overall, this project proved to be a success, and I was able to collect vast amounts of data on my competitors without having to manually search for them.

Whack a Mole

Designed and Manufactured Whack a Mole

A couple of friends and I designed and manufactured this whack a mole game. I was responsible for the eletrical as well some 3-D CAD design (basically everything). The electrical system is quite neat. We were limited to using only 1 arduino Uno, which only had a limited amount of pins. However, we wanted to operate 8 buttons, that needed to be uniquely identified. We had to individually light up each button, as well as reading each signal from the button. First, to detect which button was pressed, we used numerous resistors with known resistance and was able to read the voltage drop across the resistor. This allowed us to uniquely identify each button. Next, we had to light up each button. We used a 8 bit shift register to control the 8 LEDs. The shift register was controlled by the arduino and was able to light up each LED individually. The arduino was also connected to a speaker, which played a sound when the game was started. Overall, this project was a success, and we were able to create a fun game that was enjoyed by many.
